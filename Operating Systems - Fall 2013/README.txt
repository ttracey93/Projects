The OSPaper is an analysis of a software concept. The concept is ILR, instruction location
randomization, a concept that looks to make hacking of executable code impossible
by randomly placing each binary instruction throughout main memory. In the paper, we discuss
the implications of the concept, and both its projected strengths and weaknesses. 

The ProducerConsumerTest is a project that was made to implement synchronized java functions and deal
with multithreading and multiple threads accessing the same objects simultaneously.
